# AssetFlow

AssetFlow is a playground application created to experiment with different backend and frontend technologies in one coherent project.

The project is not intended to be a production-ready system. Its main purpose is learning, testing architectural approaches and comparing different solutions in practice.

Planned areas include:

- REST API written in Go
- Vue and PrimeVue frontend
- dictionary module
- relational data and larger data tables
- dynamic filtering and saved table configuration
- Redis caching
- BFF endpoints
- GraphQL experiments
- runtime contract validation
- mocked API responses
- Web Components integration

The scope and business domain may change during development as new technical concepts are introduced and tested.
